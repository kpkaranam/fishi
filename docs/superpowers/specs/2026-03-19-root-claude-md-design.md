# Root CLAUDE.md Priority Handling

**Date:** 2026-03-19
**Status:** Approved
**Scope:** `packages/core` (conflict-detector, merge-strategies, scaffold)

## Problem

Claude Code reads root `CLAUDE.md` at higher priority than `.claude/CLAUDE.md`. When a brownfield project has a root `CLAUDE.md` (e.g., FrAIday), FISHI's framework rules written to `.claude/CLAUDE.md` get lower priority and may be overridden by project instructions. FISHI's orchestration rules need to be at the highest priority level.

## Design

### Detection
- `conflict-detector.ts` adds `rootClaudeMdExists: boolean` to `ConflictMap`
- New conflict category `'root-claude-md'` with file `['CLAUDE.md']`
- When root CLAUDE.md exists, the `'claude-md'` category (`.claude/CLAUDE.md`) is suppressed — no prompt for it

### Merge Strategy: Prepend
New function `mergeClaudeMdTop(existing, fishiContent)`:
- FISHI section goes at the **top** with sentinel markers:
  ```markdown
  <!-- FISHI:START — Auto-generated by FISHI. Do not edit between markers. -->
  ### FISHI Framework
  [FISHI orchestration content]
  <!-- FISHI:END -->
  ---

  [existing project content unchanged below]
  ```
- Idempotent: on re-merge, replaces content between markers only
- Existing content below `---` separator is never modified

### Scaffold Logic
- If root `CLAUDE.md` exists AND resolution is `'merge'`:
  - Read root `CLAUDE.md`, prepend FISHI section via `mergeClaudeMdTop()`
  - Skip creating `.claude/CLAUDE.md`
- If root `CLAUDE.md` exists AND resolution is `'skip'`:
  - Don't touch root `CLAUDE.md`
  - Skip creating `.claude/CLAUDE.md`
- If root `CLAUDE.md` exists AND resolution is `'replace'`:
  - Replace root `CLAUDE.md` with FISHI content only
  - Skip creating `.claude/CLAUDE.md`
- If only `.claude/CLAUDE.md` exists → current behavior (append merge)
- If neither exists → create `.claude/CLAUDE.md` (greenfield)
- Non-interactive `--merge-all` → merge into root CLAUDE.md

### Backup
Root `CLAUDE.md` is included in backup when it exists (already handled by backup-manager since it backs up all conflicting files).

## Files Changed
1. `packages/core/src/generators/conflict-detector.ts` — add `rootClaudeMdExists`, add `'root-claude-md'` category, suppress `'claude-md'` when root exists
2. `packages/core/src/generators/merge-strategies.ts` — add `mergeClaudeMdTop()` function
3. `packages/core/src/generators/scaffold.ts` — root CLAUDE.md logic in config writes
4. `packages/cli/src/commands/init.ts` — prompt for root-claude-md category
5. Tests for all changes

## Testing
- Unit: `mergeClaudeMdTop()` prepends correctly, is idempotent, preserves content below
- Unit: conflict detector returns `rootClaudeMdExists: true` and `root-claude-md` category
- Unit: when root exists, `claude-md` category has 0 conflicts
- Integration: scaffold with root CLAUDE.md + merge resolution → root file has FISHI at top
- Integration: scaffold with root CLAUDE.md + skip resolution → both files untouched
- Regression: greenfield path unchanged
